4. SEGURIDAD Y AUTENTICACIÓN
Mecanismo:
- JWT Bearer
- Stateless
- Sin OAuth2

SecurityFilterChain (config/SecurityConfig.java):
- csrf disabled
- cors enabled (origen localhost:4200)
- sessionCreationPolicy STATELESS
- JwtAuthenticationFilter antes de UsernamePasswordAuthenticationFilter
- httpBasic disabled
- formLogin disabled
- permitAll: OPTIONS /**, POST /api/v1/auth/login, POST /api/v1/auth/pre-check, GET /api/v1/health
- /api/v1/admin/** -> ROLE_ADMIN
- GET /api/v1/professor/sessions/** -> ROLE_PROFESOR
- /api/v1/professor/** -> ROLE_PROFESOR o ROLE_ADMIN
- POST /api/v1/simulations -> ROLE_PROFESOR o ROLE_ADMIN
- GET /api/v1/student/activities -> ROLE_ESTUDIANTE
- GET /api/v1/clinical-cases/** -> ROLE_ESTUDIANTE/ROLE_PROFESOR/ROLE_ADMIN
- POST/PUT/DELETE /api/v1/clinical-cases/** -> ROLE_PROFESOR/ROLE_ADMIN
- /api/v1/sessions/** -> ROLE_ESTUDIANTE/ROLE_PROFESOR/ROLE_ADMIN
- anyRequest authenticated

Usuario autenticado en controllers:
- Principalmente con @AuthenticationPrincipal UserPrincipal
- En /auth/me se usa Authentication y casteo a UserPrincipal

Frontend seguridad:
- Guard root-session-redirect.guard.ts
- Guard role-authorization.guard.ts
- Interceptor auth-token.interceptor.ts agrega Authorization Bearer
- 401/403 con limpieza automática solo en request /auth/me
